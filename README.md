```16/07/2025```

I would be lying if I said that the goal of this project was to develop a lightweight and modular 64-bit operating system from scratch using Assembly and C languages, with a completely command-line interface and no graphical interface. This turned out to be more of a challenge that I read to myself. In this way, I thought I would do my childhood idea as a hobby project and gain experience in Assembly and C languages. (It caused a lot of blood, sweat, tantrums and tears.)

The idea of this system, which I call “ChallengerOS64”, is basically based on Ubuntu Server or MS-DOS. Or rather, I like this terminal look...

The development process was a complete apocalypse to tell the truth. Lots of gen-ai usage, lots of code review, lots of hours watching youtube videos about this topic and https://wiki.osdev.org/ articles that I generally don't understand.

Anyway, I'm not Terry Davis or Linus Torvalds, this is over and it's up to you. You can do whatever you want to do, I'm just going to talk a little bit about the development process.

I don't know when I will publish the code and when I will add the project details to README.md, we'll see. The reason why I haven't released it yet is that I have only poor quality code that I don't want to commit and no tutorials. ChallengerOS is a challenge, but my goal is always to spread knowledge.
